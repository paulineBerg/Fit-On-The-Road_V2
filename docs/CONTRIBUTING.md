# CONTRIBUTING -- Fit On The Road

## Branch Strategy

-   main → production-ready
-   develop → active development
-   feature/\* → new features
-   fix/\* → bug fixes

------------------------------------------------------------------------

## Workflow

1.  Create feature branch
2.  Commit changes
3.  Run: npm run lint npm run test npm run build
4.  Open Pull Request

------------------------------------------------------------------------

## Code Standards

-   TypeScript strict mode
-   1 H1 per page
-   SEO meta required for every page
-   No tracking before user consent
-   Lazy load non-critical sections

------------------------------------------------------------------------

## Documentation

Any structural, SEO, routing, or business change must update:

/docs

------------------------------------------------------------------------

## Testing

Vitest (unit tests) No E2E yet (planned)
